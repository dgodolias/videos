  • 3. Markdown vs HTML, ίδιο PR review

    Ο πιο γρήγορος τρόπος να καταλάβεις τι αλλάζει: δες το ίδιο review και στις δύο μορφές.

    Markdown

    # PR Review #142

## Issues
- auth.ts: missing error handler (high)
- db.ts: N+1 query (high)
- utils.ts: dead code (low)
- README.md: outdated example (low)

## Score
- 2 high, 2 low

    HTML artifact

    PR Review #142
    auth · db · utils · README
    HIGH · auth.ts HIGH · db.ts
    LOW · utils.ts LOW · README
    + try { await handle() } catch (e) { … } - await handle()
    Score 4/10

  • 4. Πόσο κοστίζει σε tokens

    Empirical PR-review test από τον Pasquale Pillitteri (2026):

    FormatTokens% του 1M context
    Plain Markdown1.1400,11 %
    Lean semantic HTML2.7600,28 %
    Full HTML (CSS, badges, diff)5.4800,55 %

    Δηλαδή πενταπλάσιο σε absolute terms, αλλά μόλις μισό τοις εκατό του Opus 4.7 context window. Η αντίρρηση «too expensive» δεν στέκει για final deliverables.

  • 5. Ο αντίλογος

    Η HN κοινότητα δεν αγόρασε ομόφωνα. Τα πιο δυνατά counters:

    «By gravitating to HTML you lose the ability for a human (you!) to easily co-author the document with the LLM. If it's just an explainer for your consumption, that's not a concern — but if it's a spec sheet for something more complex, I deeply value being able to dive in and edit what is produced for me.» tmhrtly, Hacker News · thread
    «Have we gone full circle? From the invention of HTML to the rebirth of plain text with MD to rediscovery of HTML.» ocimbote, Hacker News

    Η πιο συμπυκνωμένη διατύπωση από AI Architects: «Markdown for agents, HTML for humans.» Όταν το output πάει σε άλλο model (agent chain), το Markdown είναι φθηνότερο και ευκολότερο να παρσάρει. Όταν πάει σε άνθρωπο, το HTML δίνει την επιπλέον πληροφορία που το post συζητάει.

  • 6. Δοκίμασέ το

    Άνοιξε το Claude (chat ή Claude Code) και δώσε αυτό:

    Φτιάξε μου ένα HTML artifact για το παρακάτω: <τι θέλεις, π.χ. PR review, dashboard, weekly report>.

Όχι plain bullets. Θέλω color-coded sections, badges για severity, και ένα μικρό chart όπου ταιριάζει. Self-contained <style>, χωρίς external CDN.

    Σύγκρινε το result με το αντίστοιχο σε Markdown — θα δεις άμεσα τι αναφέρει ο Thariq.
